#### 第四步：保存并预览
写完之后保存文件。由于项目底层的 `posts.server.js` 会自动扫描 `app/routes/` 目录下所有 `articles.*.mdx` 文件，所以你**完全不需要修改任何其他代码**！

以后每次你想把本地的改动保存并同步到 GitHub，只需要在你的 cmd 命令行里（确保路径在项目文件夹下），按顺序运行以下三条命令：

🛠️ 日常同步代码的“Git 三连”
第一步：把所有改动放进“暂存区”（就像把商品放进购物车）

DOS
git add .
(注意：add 后面有一个空格和一个英文句号 .，代表添加当前目录下的所有变动。)

第二步：为这次改动写个备注并打包（就像结账并贴上快递单）

DOS
git commit -m "发布了我的第一篇中文博客并优化了标题字体"
(注意：双引号里面的文字你可以随便写，最好能简短说明你这次修改了什么，方便以后回顾。)

第三步：推送到 GitHub 云端（就像快递车发车）

DOS
git push
